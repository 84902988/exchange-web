## 3. 系统架构概览

系统整体采用前后端分离与异步任务协同架构：

1. 前端 Web 层负责用户交易界面、行情展示、市场列表、合约图表、用户中心与后台管理页面展示。
2. 后端 API 层负责业务接口、后台管理接口、权限校验、订单与资产相关业务入口。
3. 数据库层负责用户、订单、成交、资产、后台配置、权限、审计与业务状态持久化。
4. Redis 层用于公共行情缓存、任务队列协同、热点数据读取与部分防击穿能力。
5. Worker / Scheduler 层负责提现确认、归集、补 Gas、分红发放、佣金发放、邮件验证码、行情维护、风控 loop、强平扫描与止盈止损扫描等异步任务。
6. 后台管理层通过 RBAC 控制不同角色的操作权限，覆盖资产、交易、行情、CMS、活动、Banner、管理员与角色管理。

架构特点：

- 交易展示、后台管理、异步任务和资金链路职责分离。
- 高风险后台写操作接入权限校验。
- 行情数据采用公共缓存与最近有效数据保护，避免用户私有数据进入公共缓存。
- 资金相关路径强调状态保护、幂等保护、dry-run 与真实交易哈希区分。
