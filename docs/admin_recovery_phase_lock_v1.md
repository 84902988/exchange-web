# 后台恢复阶段锁定 V1

## 一、阶段背景

本阶段用于收口后台恢复工作。此前后台存在多类问题：

- 假数据与旧占位页仍在部分入口展示。
- 多个页面因查询结果字段缺失出现 `INTERNAL_ERROR`。
- `admin_queries.py` 中部分查询函数被 stub / empty page 兜底替代，导致页面 200 但真实数据为空。
- 多链与配置化改造后，部分查询字段、JOIN 条件、状态口径与真实表结构不一致。
- 后台模板与菜单存在乱码文案，影响运营识别与日常排查。

本阶段只恢复后台真实查询与展示稳定性，不改变交易、资金、提现、归集、分红、BD、邀请等核心业务逻辑。

## 二、恢复范围

### 用户 / 资产

- 用户管理恢复真实 `users` 查询，兼容 `user_profiles` 缺失场景。
- 资产查询恢复真实 `user_balances` 查询，保留非零余额默认语义。
- KYC 页面完成回归检查。

### 现货

- 现货订单恢复真实 `orders` 查询。
- 现货成交恢复真实 `trades` 查询。
- 查询通过 `trading_pair_id` 关联 `trading_pairs`，不再依赖不存在的 `symbol` / `user_id` 字段。

### 合约

- 合约账户恢复真实 `contract_accounts` 查询。
- 合约订单、成交、持仓恢复真实查询。
- 合约品种配置与强平记录页面完成回归检查。

### 资金

- 充值记录恢复真实 `deposits` 查询。
- 提现记录与出金审核恢复真实 `withdraw_logs` 查询。
- 站内划转与资金流水恢复真实查询。
- 页面保持分页、筛选与空状态展示。

### VIP / 分红 / BD / 邀请

- VIP 用户、等级、费率偏好页面恢复真实查询。
- 分红配置、分红池、分红统计、分红 job logs 完成恢复与回归检查。
- BD 申请、BD 账户/团队、BD 分佣、BD job logs 完成恢复与回归检查。
- 普通邀请关系与邀请分成完成恢复与回归检查。

### 配置 / 平台控制

- 资产配置、链配置、币种-网络配置页面完成回归检查。
- 交易对配置页面完成回归检查。
- 平台账户、平台调账、调账记录完成回归检查。
- Dealer 风控配置与命中日志完成回归检查。

### White-Label / 公告

- 站点配置完成回归检查。
- 首页 Banner、活动 Banner、活动管理完成回归检查。
- 公告列表、新增、编辑页面完成回归检查。
- 上传入口未做真实上传，仅确认不影响现有入口逻辑。

### 股票锁仓

- 股票锁仓配置恢复真实 `stock_token_lock_configs` 查询。
- 用户股票锁仓批次恢复真实 `user_stock_token_locks` 查询。
- 股票释放日志恢复真实 `stock_token_release_logs` 查询。
- 锁仓详情页恢复真实详情查询。

### 归集 / Gas / Job Logs

- 归集批次恢复真实 `collection_batches` 查询。
- 归集任务恢复真实 `collection_tasks` 查询。
- Gas 任务恢复真实 `gas_tasks` 查询。
- BD commission job logs 与 dividend job logs 完成真实查询回归。

## 四、当前后台能力

当前后台状态定义为 V1 恢复完成态，具备以下能力：

- 可查询：核心业务表、运营表、资金表、订单表、配置表可以通过后台页面查询。
- 可运营：已恢复页面可支持基础运营查看、筛选、分页与配置入口展示。
- 可排查：资金、订单、合约、BD、分红、归集与 Gas 任务页面可用于定位常见问题。
- 可配置：资产配置、交易对、平台调账、Dealer 风控、White-Label、Banner、公告等配置页面可打开并展示现有数据。

当前后台不定义为：

- 完整风控后台。
- 完整审计后台。
- 完整 worker / RQ 运维后台。
- 完整实时监控系统。

## 六、阶段结论

后台 V1 恢复与收口已完成。

当前后台已从假数据、占位页、查询断裂与乱码状态恢复到可查询、可运营、可排查、可配置的稳定基础状态。后续如继续建设，应以新增 V2 功能或专项优化形式推进，不再混入本轮 V1 恢复范围。
